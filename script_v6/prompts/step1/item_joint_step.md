## 🎯 总目标（必须同时满足）
1. **逐字一致**：每个 item 的 `text` 必须是原文的连续子串；所有 item 的 `text` 按顺序拼接后必须与原文完全一致（允许忽略空白差异，但不能改变任何非空白字符）。
2. **覆盖完整**：原文中每一个字符都必须且只能落在一个 item 里（不重叠、不遗漏）。
3. 模板是为了更好的表达 item 中的 text。

### ⭐ 默认策略：少镜优先（极其重要）
- **默认合并**：只要相邻片段没有命中下文 **🔴 强信号**，就应合并为**更少**的 item；不要为了「好做画面」而一句一镜。
- **句号、问号、感叹号、换行**：**单独不构成**切镜理由；连续短句若叙事角色相同、画面母题相同（同属铺垫、同属举例、同属收束），**优先合成 1 个 item**。
- **CENTER_FOCUS 链**：若多句连续都是平缓叙述、默认模板都会是 `CENTER_FOCUS`，除非其中某句是**独立金句/强钩子/强转折**，否则**应合并为 1 个 item**（合并后总长一般在 **20～55 字**为佳；超过约 **52 字**再按强信号拆，不要刚到 50 字就机械切）。
- **目标镜数（软参考）**：同一场景内，item 数量宜明显**少于**句号数量；若几乎「一句一条」，默认视为过碎，须回到本段规则重做合并检查。

## ✂️ 分镜原则

### Phase 1：时长锚点（量化判断基准）
- 正常语速约 **4~5 字/秒**；衡量对象是**整个 item 的 `text`**，不是单句：合并后整段理想 **3～10 秒（约 15～45 字）** 亦可接受，好过大量 8～18 字的碎镜。
- ⚠️ **< 12 字**的 item：除非是独立金句/强转折句，否则**禁止单独成镜**，必须并入相邻 item。
- ⚠️ **> 52 字**的单条 item：观众注意力易衰减，在**确有 🔴 强信号**的边界处拆分；不要为凑字数在中间硬拆。
- 在 `estimatedSeconds` 字段中标注该 item 全文预估秒数（字数 ÷ 4.5，保留一位小数），用于自检合并后是否仍落在合理时长。

### Phase 2：切分信号强度模型
按叙事信号强度决定是否新建镜头，**严禁按标点机械切分**：

**🔴 强信号（应新建镜头）**：
- 叙事角色彻底转换：说理/定义 → 举例/场景还原
- 叙事角色彻底转换：铺垫/铺陈 → 行动号召/结论
- 情绪段位跳变：平静分析 → 震惊转折 → 收束共情
- 视觉主体完全更换：讨论 A 话题 → 转到独立的 B 话题

**🟡 中等信号（倾向合并，但单段超过 40 字时可拆）**：
- 同一论点下的多个分点（第一、第二、第三……）
- 同一比喻的铺垫与揭示
- 同一因果链的前提与结论

**🟢 弱信号（严禁作为切分依据）**：
- **仅因句末「。」「？」「！」停顿**：只要后一句仍是同一叙事目的、同一画面母题，就必须与上一句留在同一 item。
- 普通逗号、顿号、冒号后的补充说明
- 同一句话的前后半段
- "而且""此外""所以"等常见连接词（需配合叙事角色判断）

### Phase 3：合并检查（输出前必须执行）
在输出 JSON 前，对当前 items 列表**从头到尾扫一遍**，能合并则合并：
- 相邻两条若合并后总字数 **≤ 52**，且**未**同时满足任一 🔴 强信号边界 → **必须合并为一条**。
- 相邻两条叙事角色相同、画面视觉重心相同、且模板预期均为 `CENTER_FOCUS`（或同类弱结构）→ **必须合并**；若合并后总长超过约 52 字且**中间**存在清晰 🔴 强信号，可切成 2 条，**不得**再按句号继续切碎。
- 单个建议/方法内部是 `标题 + 解释句 + 原因句/结果句` → **必须一个 item**，禁止内部分句成多条。
- 不同建议 / 不同方法之间，若已形成独立叙事动作或独立画面目标 → **不要**为统一模板强行合并。
- 若仍拆分同一建议组，必须在 `reasoning` 中写明对应 **🔴 强信号类型**（如：视觉主体已更换、情绪段位跳变等）；否则视为违规拆分，应合并回去。

### 模板选型与拆分的次序（避免为模板而拆）
- **先**按少镜原则定 item 边界，**再**为每个 item 选模板。
- 仅当**单条 item 过长**或**同一 item 内两种模板结构硬冲突**时，才为适配 `STEP_LIST` / `METHOD_STACK` 等去**调整边界**；禁止「为了用上某模板」把本可合并的平缓叙述强行切碎。

## 🧭 结构识别 → 模板选型（从上到下命中第一条）

### A. 数据/量化结论（expressionTag=data）
- 两项指标同框对比（涨跌、前后、A/B）→ `STAT_COMPARE`（必须能抽象出两个非负整数）
- 单指标完成度/占比/达成率 → `PROGRESS_RING`
- 单个核心数值/排名/增长率强调 → `KPI_HERO`

### B. 对错/强导向（expressionTag=contrast）
- 明确“别这样/要这样”“错误/正确” → `DOS_AND_DONTS`
- “不是…而是…” → `COGNITIVE_SHIFT`

### C. 多分点/清单/步骤（expressionTag=list|steps|method）
- 两方/两方案并列（更中立的 A vs B）→ `SPLIT_COMPARE`
- 当前 item 原文本身就明确展开 2～5 个并列分点/并列例子 → `LIST_MULTI_GROUP`
- 明确步骤序列/短清单，且【同一 item 内】可抽取出 2～6 条“短步骤”（如同时出现“第一/第二/第三…”或“1)/2)/3)”），并且每条步骤建议 ≤10 字、无长解释句 → `STEP_LIST`
  - 反例：只有“第一/第二”这一个标题句（单条）不要选 `STEP_LIST`（至少需要 2 条）
  - 反例：每条方法/步骤后面还跟较长解释、追问或补充句（例如任一解释段 >30 字）→ 不适合 `STEP_LIST`；优先在**单条 item 内**用 `METHOD_STACK` 或 `CENTER_FOCUS` 承载，**不要轻易**为多模板而增加镜数
- “一个标题式核心（方法/建议/观点/卖点/亮点/推荐理由）+ 2～4 句解释展开” → `METHOD_STACK`

### D. 节拍递进（expressionTag=beat）
- 同一镜头内 2～4 段口播节拍递进（问→驳→收束；铺垫→转折→结论），希望画面跟着分段变化 → `BEAT_SEQUENCE`
  - **提示**：递进里若夹杂「强对照/命名」，**优先**拆成多个 item；但若整段仍短（合并后少于约 40 字）且对照较弱，可保留单 item 另选更贴切的单一模板，避免为凑模板而碎镜。

### E. 引用/命名/聚焦（expressionTag=quote|concept|focus）
- 名言、著作、研究、媒体引用、用户评价、证言摘录 → `QUOTE_CITATION`
- 对“专业术语、概念、产品名、品牌关键词、功能名等”新名词进行解释→ `CONCEPT_CARD`
- 需要把注意力锁定到一个关键词/短语/关键细节/核心发现上 → `MAGNIFYING_GLASS`
  - 必须存在明确落点句（如“关键在于…”“核心是…”“结论是…”）
  - 需要后续可提取 1～2 个高价值锚点短语（否则不要选）

### F. 金句冲击（expressionTag=focus）
- 全片级短促强结论、无需配图 → `TEXT_FOCUS`

### G. 时间轴，先后关系（expressionTag=timeline）
- 明确时间先后/阶段演进（过去→现在→未来）→ `TIMELINE`

### H. 默认（expressionTag=neutral）
- 平缓叙述、单图即可、无上述结构 → `CENTER_FOCUS`
