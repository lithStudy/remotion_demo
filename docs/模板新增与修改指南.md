# 模板新增与修改指南

本文说明在本仓库中**新增**或**修改** Remotion 白底模板（`BW*`）时需要同步维护的代码与约定，避免前端、脚本流水线与 AI 撰稿提示脱节。

---

## 1. 模板在工程中的位置

| 层级 | 路径 / 说明 |
|------|-------------|
| 实现文件 | `src/components/templates/*.tsx`（每个文件通常包含一个 `BW*` 组件 + `templateMeta`） |
| 聚合导出 | `src/components/templates/index.ts` |
| 对外兼容导出 | `src/components/BWLayouts.tsx`（再导出给 `src/components/index.ts`） |
| 可视化总览 | `src/remotions/templateShowcase/TemplateShowcase.tsx` 中的 `SHOWCASE_SEGMENTS` |
| 公共类型与动画 | `src/components/templates/shared.ts`、`TemplateContentRenderer.tsx` |
| 脚本注册表 | `script_v5/template_registry.py` **运行时扫描**上述 TSX，**无需手改 JSON** |

---

## 2. 新增模板：推荐步骤

### 2.1 新建 TSX 文件

1. 在 `src/components/templates/` 下新增文件（命名与现有风格一致，如 `MyFeature.tsx`）。
2. 实现 React 组件（导出名建议 `BW` 前缀，如 `BWMyFeature`），遵循 Remotion 规则：**确定性渲染**（禁用 `Math.random`、避免非帧驱动的副作用）、动画用 `useCurrentFrame` / `interpolate` / `spring` 等。
3. 若需字幕、TTS、顶层锚点列表，在布局中组合 **`TemplateContentRenderer`**，并扩展 `TemplateBaseProps` / `TemplateAnchorsProps`（见 `shared.ts`）。
4. 图片入场效果优先复用 **`useImageEnterStyle`** 与类型 **`ImageEnterEffect`**，与 `param_schema` 中的 `enum` 保持一致。

### 2.2 编写 `templateMeta`（必填）

在同一文件顶部（或组件旁）导出**字面量对象**，且必须能被 Python 扫描解析：

```ts
export const templateMeta = {
  "name": "MY_TEMPLATE",           // 大写下划线，与 scene item 的 template 字段一致
  "componentExport": "BWMyFeature", // Step4 生成代码时用；缺省会回退 BWCenterFocus 并打印告警
  "description": "适用…\n差异…\n慎用…\n参数…",  // 供 Step1 AI 选型与人工阅读
  "psychology": "一句话心理学标签",
  "image_count": 1,                // 或 "2-5" 等区间字符串，供汇总表展示
  "param_schema": {                // 根节点必须是 type: "object"
    "type": "object",
    "properties": { /* ... */ },
    "required": ["…"]
  },
  "example": {
    "template": "MY_TEMPLATE",
    "param": { /* 与 param_schema 一致的示例，供 AI 看形态 */ }
  },
} as const;
```

**扫描器约束**（`script_v5/template_registry.py`）：

- 文件中必须包含**精确子串** `export const templateMeta = `（注意空格）。
- `templateMeta` 对象经去尾随逗号、去掉 `as xxx` 断言后须能被 **`json.loads`** 解析；避免在对象里写注释、模板字符串、非 JSON 字面量。
- 注册表以 `name` 为 key；写入注册表的条目会**去掉** `name` 字段，其余原样保留。

**可选字段**（与 `scene_script_validate.py` / `shared.ts` 中 `TemplateMeta` 对齐）：

| 字段 | 作用 |
|------|------|
| `content_min_items` / `content_max_items` | `item.content` 条数告警区间 |
| `content_anchor_required` | 为 `true` 时要求 `param.anchors` 非空（校验告警） |
| `param_array_sync_content` | 字符串字段名：要求 `param[该字段]` 数组长度与 **`item.content`** 条数一致（如 `BEAT_SEQUENCE` 的 `stages`） |

`param_schema` 支持的类型子集见 `shared.ts` 的 `ParamSchema` 与 `script_v5/param_schema_tools.py`；图片提示词字段使用 `"format": "image_prompt"`。

### 2.3 注册 TypeScript 导出

按顺序修改：

1. **`src/components/templates/index.ts`**：`export { BWMyFeature } from "./MyFeature"` 及对应 `Props` 类型（若有）。
2. **`src/components/BWLayouts.tsx`**：在 `export { … } from "./templates"` 中加入 `BWMyFeature` 与类型。

否则 Remotion 侧与其他文件 `from "../../components"` 的导入会找不到新组件。

### 2.4 展示用 Composition（建议）

在 **`TemplateShowcase.tsx`** 的 `SHOWCASE_SEGMENTS` 末尾追加一项：

- `key` 使用短横线命名。
- `content` 内用 `staticFile('images/template/...')` 等真实资源演示；可加 `BWSubtitle` 说明模板名与用途。
- 总时长由 `TOTAL_DURATION_TEMPLATE_SHOWCASE = SHOWCASE_SEGMENTS.length * SEGMENT_FRAMES` **自动计算**，无需改 `Root.tsx` 的 `durationInFrames`，除非单独调整 `SEGMENT_FRAMES`。

### 2.5 Python 流水线自检

在 `script_v5` 目录下重新执行或导入一次依赖注册表的步骤（如运行 `python template_registry.py` 打印指南，或跑 Step1/校验脚本），确认新模板出现在生成的指南中且无 JSON 解析跳过。

**`scene_script_validate.py`** 会对每个 item 做：未知模板回退、`param` 按 `param_schema` 校验（`content_index` 与 **`item.content`** 长度对齐）、**`item.content`** 兜底、`anchors` 与 `audioEffect` 合法值等。

### 2.6 Step4 代码生成（嵌套图片时重点核对）

`script_v5/step4_generate_remotion.py` 对以下字段有特殊处理：

- 顶层 `IMAGE_PARAM_FIELDS`（如 `imageSrc`、`leftSrc`、`rightSrc` 等）→ `staticFile(...)`。
- `images` 数组 → 每项 `src` 包装 `staticFile`。
- `stages`（节拍模板）→ 每项 `imageSrc` 包装 `staticFile`。

若新模板的图片路径出现在**更深层的嵌套结构**中，且当前走通用 `json.dumps` 分支，生成出的 TSX **可能不会**自动包 `staticFile`。此时需要**扩展** `_param_to_jsx_props`（或统一抽象一层递归处理 `format: image_prompt` 字段），与 Step2 出图路径约定一致。

## 3. 修改现有模板

1. **改视觉/动画**：只改对应 TSX 内组件逻辑；注意保持帧驱动与现有 props 兼容，或同步更新调用方。
2. **改参数字段**：同时更新：
   - 组件的 TypeScript Props；
   - `templateMeta.param_schema` 与 `example`；
   - 若字段名属于 Step4 图片白名单或需嵌套处理，同步 **`step4_generate_remotion.py`**。
3. **改 `name`**：等同于新模板 + 废弃旧名；需全局检索 `template` 字符串（含 JSON 样例、文档、旧 scene-scripts），并确认 Step1 提示词中的模板名列表由注册表自动生成即可。
4. **改 `componentExport`**：必须与 `BWLayouts` / `index.ts` 中实际导出名一致，否则 Step4 映射失败。

---

## 4. 公共约定速查

- **`TemplateBaseProps`**：`content?`、`audioSrc?`、`totalDurationFrames?`、`children?`、`style?`。
- **`ContentItem`**：`text`、`startFrame`、`durationFrames`（Step3 后对象格式）。
- **`AnchorItem`**：`text`、`showFrom`（对应 `content` 下标）、可选 `color`、`anim`、`audioEffect`。
- **字幕与锚点**：统一由 `TemplateContentRenderer` 处理时，勿重复实现一套时间轴逻辑。
- **静态资源**：Composition 内使用 `staticFile()`；`public/` 下路径与构建输出一致。

---

## 5. 检查清单（新增模板时勾选）

- [ ] TSX 中 `export const templateMeta = ` 可被 `json.loads`（无非法 TS 语法混在对象里）。
- [ ] `name` 与 `example.template` 一致；`componentExport` 与导出名一致。
- [ ] `templates/index.ts` 与 `BWLayouts.tsx` 已导出。
- [ ] `TemplateShowcase` 已追加片段（若需设计验收）。
- [ ] 运行校验流水线或 `template_registry`，确认注册表中存在该模板。
- [ ] 若含嵌套图片字段，已确认 Step4 生成结果中 `staticFile` 正确。
