# 横竖屏双导出与字幕改造指南

本文档总结已在「认知偏见 · 锚定效应」中落地的方案，供其他条目的 Remotion 动画按相同模式改造时对照使用。

---

## 1. 目标与适用场景

| 交付物 | 画布 | 用途 |
|--------|------|------|
| 横屏主片 | **1920×1080** | B站、YouTube 等全屏横屏 |
| 竖屏版 | **1080×1920** | 抖音等竖屏：中间为与横屏一致的 16:9 内容区，**上下黑边**承载标题区、进度条等，而不是浪费的纯黑 |

**设计版心**：`src/components/templates/shared.ts` 中 **`BW_LAYOUT_WIDTH` × `BW_LAYOUT_HEIGHT` = 1920×1080**。场景、`RemotionLayoutMetricsProvider` 的 `value`、以及各 BW 模板的像素与字号均按此版心直接设计。竖屏仅对整块版心做 **scale** 落入中间 16:9 视口，不再沿用其它逻辑分辨率。

---

## 2. 架构模式（推荐）

```
共享正文 MainBody（封面 + TransitionSeries + 各 Scene）
        ↑
        ├── 横屏外壳：全屏渐变 + contain/cover 缩放 + Provider
        └── 竖屏外壳：顶栏标题 + 中间 16:9 播放区 + 底部进度 + Provider
```

- **双 Composition**：在 `Root.tsx` 为同一选题注册两条——横屏一条、竖屏一条，`duration` / `schema` / `defaultProps` 一致，仅 `width`/`height` 与 `component` 不同。
- **共享正文**：抽成独立组件（如 `XXXMainBody.tsx`），**不要**在正文里写全局 BGM（BGM 放在外壳里，避免重复或遗漏）。
- **竖屏专用壳**：标题带、播放区渐变、进度条仅竖屏需要；横屏可无或极简。

**试点参考目录**：`src/remotions/认知偏见_锚定效应/`

| 文件 | 作用 |
|------|------|
| `anchoringConstants.ts` | `sceneConfigs`、时长、`DESIGN_W/H`、横竖几何常量 |
| `AnchoringBiasMainBody.tsx` | 共用 Scene树 |
| `AnchoringBiasLandscape.tsx` | 1920×1080 横屏入口 |
| `AnchoringBiasVertical.tsx` | 1080×1920 竖屏入口 |
| `AnchoringBiasVerticalChrome.tsx` | 顶栏标题 + 进度条（可随项目复制改名） |
| `认知偏见锚定效应.tsx` | 对外 re-export（schema、时长、两入口组件、常量） |

---

## 3. 几何常量（竖屏）

在竖屏画布 **Wv=1080, Hv=1920** 下建议写死：

- **16:9 播放区高度**（宽撑满时）：`VERTICAL_PLAY_H = round(1080 × 9 / 16)`（当前为 **608**）。
- **播放区垂直居中**：`VERTICAL_PLAY_TOP = round((1920 - VERTICAL_PLAY_H) / 2)`。
- **版心缩放**：`VERTICAL_CONTENT_SCALE = VERTICAL_PLAY_H / DESIGN_H`（`DESIGN_H` 与 `BW_LAYOUT_HEIGHT` 一致，为 **1080**）。
- 播放区内：`RemotionLayoutMetricsProvider value={{ width: DESIGN_W, height: DESIGN_H }}`，子元素用 `transform: scale(VERTICAL_CONTENT_SCALE)`，`transformOrigin: "top center"`，外层 `flex` 水平居中（见 `AnchoringBiasVertical.tsx`）。

进度条放在 `VERTICAL_PLAY_TOP + VERTICAL_PLAY_H + gap` 以下，全宽 `1080`，注意底部安全区（手势条）。

---

## 4. 横屏缩放策略

- **contain（当前横屏采用）**：`LANDSCAPE_CONTAIN_SCALE = min(1920/DESIGN_W, 1080/DESIGN_H)`。当 `DESIGN_W/H` 与输出一致（均为1920×1080）时 **scale=1**，无缩放裁切；若版心临时大于输出再按公式缩小。**不建议**对口播片随意用 **cover** 裁切上下。
- **cover**：`max(1920/DESIGN_W, 1080/DESIGN_H)`，铺满1920×1080 但会裁上下或左右；仅当视觉允许裁切时再考虑。

横屏同样需要 **`RemotionLayoutMetricsProvider`** 传 `DESIGN_W`、`DESIGN_H`，与 `BWImageBreath` 等逻辑一致。

---

## 5. RemotionLayoutMetrics（必读）

当 Composition 实际像素与版心不一致（例如竖屏整段为 1080×1920，而版心为1920×1080）时，`useVideoConfig()` 的 `width/height` 与版心不符，会导致：

- `BWImageBreath` 中 `slideBottom`（依赖 `height×0.25`）等入场位移错误；
- 锚点避让位移与版心不匹配。

**做法**：

1. 使用 `src/components/RemotionLayoutMetricsContext.tsx` 中的 `RemotionLayoutMetricsProvider`。
2. 在**版心缩放容器**外包一层 Provider，`value={{ width: DESIGN_W, height: DESIGN_H }}`（与场景编写分辨率一致）。
3. `BWImageBreath` 已接入：有 Provider 时优先用覆盖尺寸算入场与避让。

其他模板若直接依赖 `useVideoConfig` 的宽高做像素动画，也需要同样处理或单独适配。

---

## 6. Root 与导出命令

1. 在 `src/Root.tsx` 增加两条 `Composition`：
   - `id` 建议：`你的条目`（横屏）、`你的条目竖屏`（竖屏）。
   - 横屏：`width={1920} height={1080}`。
   - 竖屏：`width={1080} height={1920}`。
2. 在 `package.json` 的 `scripts` 中为该选题增加两条 `remotion render`，分别指向上述 `id` 与输出路径（可参考现有 `render:认知偏见锚定效应_*`）。

---

## 7. 字幕与小屏可读（全局）

已在 **`src/components/BWPrimitives.tsx`** 的 **`BWSubtitle`** 统一调整：

- **移除**顶部灰底条（原 `position==="top"` 时的半透明黑底）。
- **移除**主序列里仅用于衬底的 **`rgba(0,0,0,0.5)`底部条**（各 `remotions/*/xxx.tsx` 里 `AbsoluteFill` 下、`TransitionSeries` 上的那块 `div`）。锚定效应正文在 `AnchoringBiasMainBody.tsx` 中已删；其他条目若仍保留该 `div`，应删掉以与全局字幕策略一致。
- 字幕样式：深字 `#0f172a`、更大字重、多层浅色外晕式 `textShadow`，减轻对灰条的依赖，便于竖屏小字号观看。

改造其他动画时：**不要再加回**底部半透明条；若某模板在深色底上对比不足，应局部改模板或给 `BWSubtitle` 增加 `style`/`variant`，而不是恢复全局灰条。

## 8. 新条目改造检查清单

复制本清单，按顺序勾掉即可：

- [ ] 抽出 `*Constants.ts`（或等价模块）：`sceneConfigs`、各类 `calculate*Duration`、`TRANSITION_DURATION`、`COVER_DURATION_FRAMES`、`DESIGN_W/H`、横竖专用常量。
- [ ] 抽出 `*MainBody.tsx`：仅封面 + `TransitionSeries` + Scenes，**无** BGM、**无** 底部灰条。
- [ ] 实现 `*Landscape.tsx`：Audio + 全屏背景 + Provider + contain 缩放 + `MainBody`。
- [ ] 实现 `*Vertical.tsx`：Audio + 顶栏（可选独立组件）+ 播放区 + Provider + 缩放 + `MainBody` + 进度区。
- [ ] 入口文件 re-export：schema、总时长、`MainBody` 不必导出给 Root，仅导出两个 Composition 用的组件 + 需要的常量。
- [ ] `Root.tsx` 注册双 Composition。
- [ ] `package.json` 增加双 `render` 脚本。
- [ ] 确认全片无 `rgba(0,0,0,0.5)` 字幕衬底 `div`（除非你有意保留的特殊场景）。
- [ ] Studio 中分别预览横屏 id与竖屏 id，检查字幕与锚点是否被裁切。

---

## 9. 已知注意点

- **竖屏预览**：必须选中带「竖屏」后缀的 Composition，否则看不到黑边壳与进度条布局。
- **脚本/自动化**：若旧流程只渲染单一 id，需改为横竖各渲染一次或按平台选择 id。
- **版心统一**：新条目与模板均以 **1920×1080** 为准；旧条目若 Composition 仍为其它尺寸，需改画布并设 `DESIGN_*` 与 Provider 与模板一致后回归。
