## 🎯 总目标（必须同时满足）
1. **逐字一致**：每个 item 的 `text` 必须是原文的连续子串；所有 item 的 `text` 按顺序拼接后必须与原文完全一致（允许忽略空白差异，但不能改变任何非空白字符）。
2. **覆盖完整**：原文中每一个字符都必须且只能落在一个 item 里（不重叠、不遗漏）。
3. **表达原子化**：**一个 item 只允许一个主表达结构**（对比/命名/步骤/引用/数据/节拍递进/金句冲击/平铺叙述）。若同一段里混了两种结构，必须拆成多个 item。
4. **长度合理**：为了避免视觉疲劳，一个item中的text拼接后如果长于 50 字应该拆分成多个item。
5. 模板是为了更好的表达item中的text

## ✂️ 分镜原则

### Phase 1：时长锚点（量化判断基准）
- 正常语速约 **4~5 字/秒**，单个镜头的理想口播时长：**3~8 秒（约 12~40 字）**
- ⚠️ **< 12 字**的分段：除非是独立金句，否则**必须合并**到相邻镜头
- ⚠️ **> 50 字**的分段：意味着画面停留太久，观众注意力衰减，建议拆分，但拆分点本身仍需有叙事依据
- 在 `estimatedSeconds` 字段中标注预估秒数（字数 ÷ 4.5，保留一位小数）
- 除非模板使用TextFocus，否则不允许content中只有一个Text的情况

### Phase 2：切分信号强度模型
按叙事信号强度决定是否新建镜头，**而非按标点机械切分**：

**🔴 强信号（应新建镜头）**：
- 叙事角色彻底转换：说理/定义 → 举例/场景还原
- 叙事角色彻底转换：铺垫/铺陈 → 行动号召/结论
- 情绪段位跳变：平静分析 → 震惊转折 → 收束共情
- 视觉主体完全更换：讨论 A 话题 → 转到独立的 B 话题

**🟡 中等信号（倾向合并，但单段超过 40 字时可拆）**：
- 同一论点下的多个分点（第一、第二、第三……）
- 同一比喻的铺垫与揭示
- 同一因果链的前提与结论

**🟢 弱信号（严禁作为切分依据）**：
- 普通逗号、顿号、冒号后的补充说明
- 同一句话的前后半段
- "而且""此外""所以"等常见连接词（需配合叙事角色判断）

### Phase 3：合并检查
拆完后对所有 items 做全局合并检查：
- 相邻两个 items 若切分信号为"弱"且合并后总字数不超过 45 字 → **强制合并**
- 相邻两个 items 若叙事角色完全相同、画面视觉重心也相同 → **强制合并**

## 🧭 结构识别 → 模板选型（从上到下命中第一条）

### A. 数据/量化结论（expressionTag=data）
- 两项指标同框对比（涨跌、前后、A/B）→ `STAT_COMPARE`（必须能抽象出两个非负整数）
- 单指标完成度/占比/达成率 → `PROGRESS_RING`
- 单个核心数值/排名/增长率强调 → `KPI_HERO`

### B. 对错/强导向（expressionTag=contrast）
- 明确“别这样/要这样”“错误/正确” → `DOS_AND_DONTS`
- “不是…而是…” → `COGNITIVE_SHIFT`

### C. 多分点/清单/步骤（expressionTag=list|steps|method）
- 两方/两方案并列（更中立的 A vs B）→ `SPLIT_COMPARE`
- 当前 item 原文明确展开 **2～4** 个并列分点/并列例子，且每点需独立配图（宫格）→ `PANEL_GRID`
- 明确步骤序列/短清单，且【同一 item 内】可抽取出 2～6 条“短步骤”（如同时出现“第一/第二/第三…”或“1)/2)/3)”），并且每条步骤建议 ≤10 字、无长解释句 → `STEP_LIST`
  - 反例：只有“第一/第二”这一个标题句（单条）不要选 `STEP_LIST`（至少需要 2 条）
  - 反例：每条方法/步骤后面还跟较长解释、追问或补充句（例如任一解释段 >30 字）→ 不适合 `STEP_LIST`，应拆分 item，并优先用 `METHOD_STACK`（标题式核心 + 解释展开）或 `CENTER_FOCUS`
- “一个标题式核心（方法/建议/观点/卖点/亮点/推荐理由）+ 2～4 句解释展开” → `METHOD_STACK`
- 同一**案例/子话题**、**单主图**，口播 **4～6 句** 呈「个案→误判/推论→纠偏→收束」叙事弧 → `CASE_BREAKDOWN`（非「第一/第二」方法清单、非多图节拍时用本模板；方法要点式仍用 `METHOD_STACK`）

### D. 节拍递进（expressionTag=beat）
- 同一镜头内 2～4 段口播节拍递进（问→驳→收束；铺垫→转折→结论），希望画面跟着分段变化 → `BEAT_SEQUENCE`
  - **提示**：如果递进里还夹杂“强对照/命名”，通常应拆成多个 item，不要硬塞一个 `BEAT_SEQUENCE`。
- 同一镜头内 **因果链 / 机制传导**（因→果→再果，每环一图，无时间轴刻度）→ `CAUSE_CHAIN`

### E. 引用/命名/聚焦（expressionTag=quote|concept|focus）
- 名言、著作、研究、媒体引用、用户评价、证言摘录 → `QUOTE_CITATION`
- 对“专业术语、概念、产品名、品牌关键词、功能名等”新名词进行解释→ `CONCEPT_CARD`
- 需要把注意力锁定到一个关键词/短语/关键细节/核心发现上 → `MAGNIFYING_GLASS`
  - 必须存在明确落点句（如“关键在于…”“核心是…”“结论是…”）
  - 需要后续可提取 1～2 个高价值锚点短语（否则不要选）

### F. 金句冲击（expressionTag=focus）
- 全片级短促强结论、无需配图 → `TEXT_FOCUS`

### G. 时间轴，先后关系（expressionTag=timeline）
- 明确时间先后/阶段演进（过去→现在→未来）→ `TIMELINE`

### G2. 收束清单（expressionTag≈conclusion|list）
- 收束、行动检查项、要点复诵，需要**逐项打勾显现**（非编号短步骤、非并列大图组）→ `CHECKLIST_REVEAL`

### H. 默认（expressionTag=neutral）
- 平缓叙述、单图即可、无上述结构 → `CENTER_FOCUS`
