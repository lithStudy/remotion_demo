# 阶段 2：逐场景分镜 + 模板选型

## 目标

在不改写口播原文的前提下，把场景原文切成"原子化"的镜头 item，并为每个镜头选择最合适的 `template`。

## 硬性约束

1. **逐字一致**：每个 item 的 `text` 必须是原文的连续子串；所有 item 的 `text` 按顺序拼接后必须与原文完全一致（允许忽略空白差异，但不能改变任何非空白字符）
2. **覆盖完整**：原文中每一个字符都必须且只能落在一个 item 里（不重叠、不遗漏）
3. **表达原子化**：一个 item 只允许一个主表达结构（对比/命名/步骤/引用/数据/节拍递进/金句冲击/平铺叙述）。若同一段里混了两种结构，必须拆成多个 item
4. **长度合理**：一个 item 中的 text 拼接后如果长于 50 字应拆分成多个 item
5. 模板是为了更好的表达 item 中的 text

## 分镜原则

### 时长锚点（量化判断基准）

- 正常语速约 **4~5 字/秒**，单个镜头理想口播时长：**3~8 秒（约 12~40 字）**
- **< 12 字**的分段：除非是独立金句，否则**必须合并**到相邻镜头
- **> 50 字**的分段：画面停留太久，观众注意力衰减，建议拆分（拆分点仍需有叙事依据）
- 除非模板使用 TEXT_FOCUS，否则不允许 content 中只有一个 text 的情况

### 切分信号强度模型

按叙事信号强度决定是否新建镜头，**而非按标点机械切分**：

**强信号（应新建镜头）**：
- 叙事角色彻底转换：说理/定义 → 举例/场景还原
- 叙事角色彻底转换：铺垫/铺陈 → 行动号召/结论
- 情绪段位跳变：平静分析 → 震惊转折 → 收束共情
- 视觉主体完全更换：讨论 A 话题 → 转到独立的 B 话题

**中等信号（倾向合并，但单段超过 40 字时可拆）**：
- 同一论点下的多个分点（第一、第二、第三……）
- 同一比喻的铺垫与揭示
- 同一因果链的前提与结论

**弱信号（严禁作为切分依据）**：
- 普通逗号、顿号、冒号后的补充说明
- 同一句话的前后半段
- "而且""此外""所以"等常见连接词（需配合叙事角色判断）

### 合并检查

拆完后对所有 items 做全局合并检查：
- 相邻两个 items 若切分信号为"弱"且合并后总字数不超过 45 字 → **强制合并**
- 相邻两个 items 若叙事角色完全相同、画面视觉重心也相同 → **强制合并**

## 结构识别 → 模板选型（从上到下命中第一条）

### A. 数据/量化结论

- 两项指标同框对比（涨跌、前后、A/B）→ `STAT_COMPARE`（必须能抽象出两个非负整数）
- 单指标完成度/占比/达成率 → `PROGRESS_RING`
- 单个核心数值/排名/增长率强调 → `KPI_HERO`

### B. 对错/强导向

- 明确"别这样/要这样""错误/正确" → `DOS_AND_DONTS`
- "不是…而是…" → `COGNITIVE_SHIFT`

### C. 多分点/清单/步骤

- 两方/两方案并列（更中立的 A vs B）→ `SPLIT_COMPARE`
- 当前 item 原文明确展开 2～4 个并列分点/并列例子，且每点需独立配图（宫格）→ `PANEL_GRID`
- 明确步骤序列/短清单，且同一 item 内可抽取出 2～6 条"短步骤"（如"第一/第二/第三…"或"1)/2)/3)"），且每条步骤 ≤10 字、无长解释句 → `STEP_LIST`
  - 反例：只有"第一/第二"这一个标题句（单条）不选 `STEP_LIST`（至少需 2 条）
  - 反例：每条方法/步骤后面还跟较长解释 >30 字 → 不适合 `STEP_LIST`，应优先用 `METHOD_STACK` 或 `CENTER_FOCUS`
- "一个标题式核心（方法/建议/观点/卖点/亮点/推荐理由）+ 2～4 句解释展开" → `METHOD_STACK`
- 同一案例/子话题、单主图，口播 4～6 句呈「个案→误判/推论→纠偏→收束」叙事弧 → `CASE_BREAKDOWN`（非方法清单、非多图节拍时用；方法要点式仍用 `METHOD_STACK`）

### D. 节拍递进

- 同一镜头内 2～4 段口播节拍递进（问→驳→收束；铺垫→转折→结论），画面跟着分段变化 → `BEAT_SEQUENCE`
  - 如果递进里还夹杂"强对照/命名"，通常应拆成多个 item
- 同一镜头内因果链/机制传导（因→果→再果，每环一图，无时间轴刻度）→ `CAUSE_CHAIN`

### E. 引用/命名/聚焦

- 名言、著作、研究、媒体引用、用户评价、证言摘录 → `QUOTE_CITATION`
- 对"专业术语、概念、产品名、品牌关键词、功能名"等新名词进行解释 → `CONCEPT_CARD`
- 注意力锁定到一个关键词/短语/关键细节/核心发现上 → `MAGNIFYING_GLASS`
  - 必须存在明确落点句（如"关键在于…""核心是…""结论是…"）
  - 需可提取 1～2 个高价值锚点短语（否则不选）

### F. 金句冲击

- 全片级短促强结论、无需配图 → `TEXT_FOCUS`

### G. 时间轴

- 明确时间先后/阶段演进（过去→现在→未来）→ `TIMELINE`

### G2. 收束清单

- 收束、行动检查项、要点复诵，需逐项打勾显现（非编号短步骤、非并列大图组）→ `CHECKLIST_REVEAL`

### H. 默认

- 平缓叙述、单图即可、无上述结构 → `CENTER_FOCUS`

## 模板语义一致性检查

分镜选型完成后，做以下自检：

- `SPLIT_COMPARE` 但口播含"升级为""而不是""不再""别再""应该"等方向性表达 → 应改为 `COGNITIVE_SHIFT` 或 `DOS_AND_DONTS`
- `CONCEPT_CARD` 的 conceptName 应在口播中真实出现（不是自己编造的术语）
- 连续多个 `CENTER_FOCUS` → 考虑是否可合并或改用更适合的模板

## 阶段 2 输出格式

对每个 scene 产出 items 数组：

```json
{
  "items": [
    {
      "text": "逐字口播片段（必须来自原文）",
      "narrativeType": "HOOK | LOGIC | CASE | DATA | CONCLUSION | TRANSITION",
      "template": "模板名（必须来自合法列表）",
      "reasoning": "一句话：该段主表达结构 → 选该模板原因 → 若有拆分/合并，理由",
      "order": 1
    }
  ]
}
```

- `order` 从 1 递增，在 scene 内唯一
- `narrativeType` 可选值：`HOOK`（钩子引入）、`LOGIC`（逻辑论证）、`CASE`（案例/举例）、`DATA`（数据引用）、`CONCLUSION`（结论/收束）、`TRANSITION`（过渡）
